# RMK data team internship 2026 test challenge
The challenge is designed to demonstrate your style, statistical thinking, documentation, and coding skills.
### challenge
> **TL;DR**
> The objective of the challenge is to create a probability scale. That is, a list of events along with their probabilities to help readers develop intuition about probability.

People have an intuitive understanding of distances. Things that are a few centimetres long can be held by hand; metres are the scale of buildings; kilometres are for distances between cities, etc.

People don't have the same intuitive understanding of probabilities. What are events that happen with probability 0.4? How about 0.004?

Your challenge is to find data about the probabilities of events, process it programmatically (write code), and present it graphically.

### data
We strongly encourage you to look for input data in the [Estonian Data Portal](https://andmed.eesti.ee/). There are a lot of data points there that can be converted to probabilities.

Ingest the input data programmatically. If that's not possible, at least include a proper citation to the source.
### code
Present your solution as a (GitHub) repository. Include work history, not just a single "miraculous" commit.

Use Python (or R, if you really have to). Make every step of the solution programmatically reproducible, i.e., use code to load data, process it, create graphics, save results, etc.

We like proper documentation. If we can't understand your code within 3 minutes, we probably won't read further.
- READMEs
- good naming
- docstrings and comments
- run instructions
- example of the end result
- license
- [etc.](https://peps.python.org/pep-0008/)
### scope
We expect that candidates will not be able to complete the challenge in a sensible amount of time. You get extra points for describing your thought process and proposed solution, even if it's not fully implemented.

You don't have to be perfect, just better than other applicants. The challenge presents many possible ways to stand out. Know thyself and choose your battles wisely!
- Come up with the most interesting events / input data.
- Use tight code and API requests.
- Add Bayesian reasoning for extra flavour.
- Do [effect sizes](https://pmc.ncbi.nlm.nih.gov/articles/PMC4592565/) or [correlations](https://www.astralcodexten.com/p/attempts-to-put-statistics-in-context) instead of probabilities.
- Come up with an innovative data presentation.
- Create an aesthetic graphical output.
- Present the story and logic behind your solution clearly.

> Note that our data team does not do much front-end work. You can create a nice front end, but don't make it the core of your solution, lest your effort goes unappreciated.

See also: [feedback to 2025 solutions](../2025/feedback.md).
### ai
Feel free to use as much AI assistance as you want.

However, a real person from our side will attempt to read your code. This person will decide your eligibility. Don't make them hate you! If you one-shot 10k lines of code with minimal human input, you might well get rejected at the first pass (even if it works).
### results
Depending on the number of applicants, we might or might not be able to give feedback to everyone (why you were selected or not). We'll let you know.

Either way, the challenge is designed to be a good portfolio item, so your time will not be wasted.
